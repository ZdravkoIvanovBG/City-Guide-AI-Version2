Add a new fifth tab to the Generated Plan page. It sits after the
Misc tab in the pill nav:

Itinerary | Hotels | Restaurants | Bonus Activities | Before You Go

This tab has three sections inside it, each collapsible with a
smooth Framer Motion height animation. All three sections are
generated by Gemini as part of the main plan generation call —
add them to the existing Gemini JSON response shape so there is
no extra API call needed.

---

## Section 1 — Packing List

### Gemini prompt addition

Add a "packingList" field to the plan JSON:

"packingList": {
  "categories": [
    {
      "name": "Documents",
      "items": [
        { "label": "Passport", "essential": true, "note": "Check expiry — must be valid 6 months beyond travel date" },
        { "label": "Travel insurance documents", "essential": true, "note": null },
        { "label": "Hotel booking confirmations", "essential": false, "note": null }
      ]
    },
    {
      "name": "Clothing",
      "items": [...]
    },
    {
      "name": "Toiletries",
      "items": [...]
    },
    {
      "name": "Electronics",
      "items": [...]
    },
    {
      "name": "Health & Safety",
      "items": [...]
    },
    {
      "name": "City-specific",
      "items": [...]
    }
  ]
}

Rules for Gemini:
- Tailor the list to the specific city, season (based on travel
  dates), trip length, and traveller type (solo/couple/family/group)
- "City-specific" category contains things unique to that destination:
  e.g. for Tokyo: "IC Card (Suica) for transit", "Portable WiFi or
  SIM card — recommended for Japan", "Pocket umbrella — rain is common"
  e.g. for Bali: "Mosquito repellent — essential", "Sarong for temple
  visits", "Cash in IDR — many places don't accept cards"
- Mark essential: true only for genuinely critical items
- Keep notes concise and practical, null if nothing to add
- Each category should have 4–8 items, not an exhaustive dump

### Frontend UI

- Each category is a collapsible section (open by default)
- Each item is a checkbox row: [ ☐ Passport ⚠ essential ] with the
  note shown in smaller text below if present
- Checked state is saved to localStorage keyed by planId so it
  persists across page refreshes
- Essential items have a small amber dot indicator
- At the top: a progress bar showing "12 of 34 items packed" that
  fills as the user checks items off
- A "Reset all" button in the top right of the section
- On mobile: full width, comfortable tap targets (min 44px row height)

## Section 2 — Budget Estimator

### Gemini prompt addition

Add a "budgetEstimate" field to the plan JSON:

"budgetEstimate": {
  "currency": "EUR",
  "currencySymbol": "€",
  "budgetTier": "mid-range",
  "dailyBreakdown": [
    {
      "day": 1,
      "date": "2025-06-03",
      "items": [
        { "category": "Entry fees", "description": "Sagrada Familia + Park Güell", "estimatedCost": 36, "notes": "Book online to avoid queues" },
        { "category": "Transport", "description": "Metro day pass", "estimatedCost": 11, "notes": "Covers all zones" },
        { "category": "Lunch", "description": "Mid-range restaurant, 2 courses", "estimatedCost": 18, "notes": null },
        { "category": "Dinner", "description": "Tapas bar in El Born", "estimatedCost": 25, "notes": "Prices rise after 8pm" },
        { "category": "Snacks & drinks", "description": "Coffee, water, snacks", "estimatedCost": 10, "notes": null }
      ],
      "dayTotal": 100
    }
  ],
  "fixedCosts": [
    { "category": "Accommodation", "description": "Mid-range hotel per night", "estimatedCostPerNight": 95, "totalEstimated": 380, "notes": "Based on 4 nights" }
  ],
  "tripTotal": {
    "low": 480,
    "mid": 580,
    "high": 720
  },
  "localTips": [
    "Lunch menus (menú del día) in Barcelona offer 3 courses for €12–15 — much cheaper than dinner",
    "Buy a T-Casual 10-trip metro card instead of single tickets to save 40%"
  ]
}

Rules for Gemini:
- Use the local currency of the destination city
- Base amounts on the user's selected budget tier (budget /
  mid-range / luxury) from the plan builder form
- Entry fees should match the real prices from the itinerary
  destinations already generated in the plan
- Daily transport cost should reflect the howToGetThere modes
  already suggested — if metro was recommended, price a day pass
- Meal costs should reflect local price levels for that city
  specifically (Tokyo ramen vs Paris bistro vs Bali warung)
- tripTotal.low / mid / high gives a realistic range accounting
  for variance in choices

### Frontend UI

- Section header: "Budget Estimator" with the selected tier badge
  (Budget / Mid-range / Luxury) and total trip range:
  "Estimated total: €480 – €720"
- Each day is a collapsible accordion row showing the day date
  and day total: "Day 1 · June 3 · ~€100"
  Expanding it shows the itemised list for that day
- Each item row: category icon + description + cost on the right
- Fixed costs (accommodation) shown in a separate card below the
  daily breakdown
- At the bottom: a clean summary card showing:
    Accommodation:   €380
    Daily expenses:  ~€200
    ─────────────────────
    Estimated total: €480 – €720
- A toggle at the top: "Per person" / "Total" — multiplies all
  values by the group size (pulled from traveller type selection)
- Category color coding: entry fees = amber, transport = blue,
  food = green, accommodation = purple — applied as left border
  color on each row
- All amounts are editable inline — user can tap any cost and
  type their own value, which recalculates the totals live
- A "Reset to estimates" button restores Gemini's original values
- Edited values saved to localStorage keyed by planId

### Gemini prompt addition

Add a "tripChecklist" field to the plan JSON:

"tripChecklist": {
  "categories": [
    {
      "name": "Before you book",
      "items": [
        {
          "label": "Check visa requirements for [country]",
          "essential": true,
          "link": "https://www.iatatravelcentre.com",
          "linkLabel": "Check visa requirements",
          "detail": "Citizens of EU, US, UK, Canada do not need a visa for stays under 90 days"
        },
        {
          "label": "Check passport expiry",
          "essential": true,
          "link": null,
          "detail": "Must be valid for at least 6 months beyond your return date"
        }
      ]
    },
    {
      "name": "Accommodation & flights",
      "items": [...]
    },
    {
      "name": "Health & safety",
      "items": [...]
    },
    {
      "name": "Money & payments",
      "items": [...]
    },
    {
      "name": "Useful apps for [city]",
      "items": [
        {
          "label": "Download Moovit or Citymapper",
          "essential": false,
          "link": null,
          "detail": "Best apps for real-time public transit in Barcelona"
        },
        {
          "label": "Download Bolt or Cabify for taxis",
          "essential": false,
          "link": null,
          "detail": "Cheaper than street taxis, widely used in Spain"
        }
      ]
    },
    {
      "name": "On arrival",
      "items": [...]
    }
  ]
}

Rules for Gemini:
- Visa and health requirements must be city/country specific
- "Useful apps" category must contain real apps that locals
  actually use in that specific city for transit, food delivery,
  taxis, etc.
- "Money & payments" should mention whether cards are widely
  accepted, if cash is needed, ATM availability, and the local
  tipping culture
- "Health & safety" should mention vaccinations if relevant,
  tap water safety, emergency numbers for that country (police,
  ambulance), and any specific safety notes for that city

### Frontend UI

- Same checkbox row design as the packing list
- Items with a link show a small external link icon after the label
  — clicking opens in a new tab
- Items with a detail show it in smaller text below the label
- Essential items have the amber dot indicator
- Checked state saved to localStorage keyed by planId
- Separate progress bar from the packing list: "8 of 22 tasks done"

## Layout of the full "Before You Go" tab

Stack the three sections vertically in this order:
1. Trip Checklist (most time-sensitive, show first)
2. Budget Estimator
3. Packing List

Each section has:
- A bold section header with an icon and an expand/collapse chevron
- All sections open by default on first visit
- Collapse state saved to localStorage keyed by planId

On mobile: full width, generous padding, large tap targets throughout.

The tab label in the floating pill nav must read exactly "Before You Go".
No other tab names should change.
